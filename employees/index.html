    Employees

    Employee is the entity that is responsible to manage all the employees associated to a User entity.

    The entity has a One to Many relationship with Experience as he can have several experiences.

    Properties

    Field Type Description
    employeeId Int The foreign key ID of the Employee that have the Experience
    trigram String The trigram of the Employee (ARI for example)
    availableDate String The date where the Employee complete a mission

    Info

    All parameters required by routes of the API need to be passed as JSON format

    Read all employees

    Returns the list of all employees and their experiences

    GET

    https://cvteque.herokuapp.com/employees
    Response example
    [
  {
    "experiences": [],
    "trigram": "PMU",
    "availableDate": "03-11-2017"
  },
  {
    "experiences": [],
    "trigram": "ARI",
    "availableDate": "03-11-2017"
  },
  {
    "experiences": [],
    "trigram": "SCH",
    "availableDate": "03-11-2017"
  }
]

    Warning

    In order to create an Employee, the corresponding User needs to be created before. This POST request requires the ID of the User.

    POST

    https://cvteque.herokuapp.com/employees
    JSON example
    {
    "trigram":"ARI",
    "availableDate": "12-06-2017",
    "userId":1
}
    Response example
    {
  "employee": {
    "trigram": "PLU",
    "userId": 1,
    "availableDate": "03-11-2017"
  },
  "success": true
}

    Update an employee

    PATCH

    https://cvteque.herokuapp.com/employees/:id
    Usage example
    https://cvteque.herokuapp.com/employees/99

    PARAMETERS

    Field Type Description
    id Int The employee's ID you want to update.
    JSON example
    {
    "trigram":"PLU",
    "availableDate":"03-11-2017",
    "userId":1
}

    Tip

    The JSON is the same as Create, just change the values you want.

    Response example
    {
  "updated object": {
    "trigram": "PLU",
    "userId": 1,
    "availableDate": "03-11-2017"
  },
  "success": true
}

    Create an experience for an Employee

    This endpoint add an Experience for an Employee.

    POST

    https://cvteque.herokuapp.com/employees/:id/experiences
    EXAMPLE USAGE
    https://cvteque.herokuapp.com/employees/99/experiences

    PARAMETERS

    Field Type Description
    id Int The employee's ID you want to add an experience.
    Requirements

    It needs a JSON Experience object passed in the request's body.

    EXAMPLE OF JSON
    {
    "title":"Vapor Developer",
    "compagnyName":"Volvo",
    "desc":"Adolescebat autem obstinatum propositum...",
    "startDate":"15-04-2015",
    "endDate":"15-08-2017"
}

    Update an experience for an Employee

    This endpoint updates an existing experience for an Employee.

    PATCH

    https://cvteque.herokuapp.com/employees/:employee_id/experiences/:experience_id
    EXAMPLE USAGE
    https://cvteque.herokuapp.com/employees/99/experiences/10

    PARAMETERS

    Field Type Description
    employee_id Int The employee's ID you want to update the experience.
    experience_id Int The experience's ID you want to update.

    Warning

    All the parameters have to be sent to update an Experience.

    EXAMPLE OF JSON
    {
    "title":"Vapor Developer",
    "compagnyName":"Volvo",
    "desc":"Adolescebat autem obstinatum propositum...",
    "startDate":"15-04-2015",
    "endDate":"15-08-2017"
}

    Tip

    The same JSON as Create an Experience, just change the values you want.
